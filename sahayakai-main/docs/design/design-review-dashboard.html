SahayakAI · Design Review Board

Finish the system you already built.

Fourteen specialists reviewed the live product independently. The verdict is unusually unanimous — and encouraging. SahayakAI already has the bones of an excellent design system: tokens, primitives, motion variables, Indic fonts, a saffron ramp, an 11-language progress engine. It just isn't wired up. Adoption sits near 30%, so the app reads like three design systems stitched at the shell. The redesign isn't invention — it's finishing and enforcing the system that exists, then elevating it.

Cross-cutting consensus

Seven findings surfaced independently across multiple lenses. When a UI designer and an accessibility engineer reach the same conclusion from opposite ends, that's not opinion — it's signal.

The token system is only half-adopted

rounded-surface-* used 6× vs ~200 raw radii · type-h* 5× vs font-headline 129× · PageShell built, used by ~0 pages. The root cause behind everything below.

Section 03

The redesign, sequenced

Five phases. Accessibility, localization, and illustration ride along as acceptance gates in every phase — never a cleanup phase at the end.

Phase 0

Foundations

Ratify tokens · fix saffron contrast · self-host fonts · token CI lint · vectorize logo.

Phase 1

Core loop

Redesign dashboard & teacher home. Generate-then-refine; demote the config form.

Phase 2

The tools

Collapse ~15 generators onto GeneratorPage; wire real SSE; add tool chaining.

Phase 3

Community

Feed, library, messages — one consistent social surface on the ratified system.

Phase 4

Marketing

Activate orphaned proof; rebrand Indic pages; commission the illustration set.

Woven through every phase  · Accessibility (contrast, 48px targets, aria-live results, read-aloud) · Localization (per-script type, .indic-text default) · Illustration (Warm Chalk-Line, decorative accent only).  Personality  warm · calm · confident · rooted · uncluttered.

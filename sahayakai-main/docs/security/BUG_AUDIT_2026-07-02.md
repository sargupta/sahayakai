## P0 — CRITICAL (fix before anything else)

### C1. Any user can self-grant platform admin AND free premium — client-side
- **Where:** [firestore.rules:39](../../firestore.rules) — `match /users/{userId} { allow read, update: if isOwner(userId); }` with **no field validation**.
- **Why it works:** [auth-utils.ts:15](../../src/lib/auth-utils.ts) — `isAdmin()` returns true when `users/{uid}.role === 'admin'`. Client-SDK writes to the user doc are live in prod ([app-sidebar.tsx:351](../../src/components/app-sidebar.tsx)), so the Firestore rule is the only gate — and it accepts arbitrary fields.
- **Exploit:** Browser console: `updateDoc(doc(db,'users',myUid),{role:'admin'})` → full platform admin (cost dashboard, log dashboard w/ PII, org provisioning). Or `{planType:'premium'}` → free premium, bypassing Razorpay. The `PROFILE_WRITABLE_FIELDS` allowlist in `profile.ts` is **bypassed** because the client writes directly, not through the server action.
- **Confirmed by:** 4 independent agents.
- **Fix:** In the `users` update rule, block privileged-field mutation:
  ```
  allow update: if isOwner(userId)
    && !request.resource.data.diff(resource.data).affectedKeys()
        .hasAny(['role','planType','plan','isAdmin','customClaims','impactScore',
                 'badges','verifiedStatus','organizationId','administrativeRole',
                 'email','uid','createdAt','razorpaySubscriptionId']);
  ```
  Move `isAdmin` off a mutable Firestore field onto a custom claim or server-side email allowlist.

### C2. Cross-tenant analytics access via self-asserted org role
- **Where:** [profile.ts:223,236](../../src/app/actions/profile.ts) — `administrativeRole` and `organizationId` are both in `PROFILE_WRITABLE_FIELDS`; [organizations/[orgId]/analytics/route.ts:50-62](../../src/app/api/organizations/[orgId]/analytics/route.ts) authorizes anyone whose `profile.organizationId === orgId` **and** `administrativeRole ∈ {principal, vice_principal}` — both attacker-controlled.
- **Exploit:** `updateProfileAction({administrativeRole:'principal', organizationId:'<any-school-id>'})`, then `GET /api/organizations/<victimOrg>/analytics` → every teacher's health scores, names, engagement, student-impact for a school you have no relationship with. Same gate opens the principal dashboard.
- **Fix:** Remove both fields from `PROFILE_WRITABLE_FIELDS`; derive org role only from the server-written `organizations/{orgId}/members/{uid}` subcollection or custom claims.

### C3. Live server secrets baked into the Docker image
- **Where:** [.dockerignore](../../.dockerignore) ends with `*.local` then `!.env.local` (re-includes it); [Dockerfile:13](../../Dockerfile) `COPY . .` in the builder stage. **Verified:** `.env.local` (3.6 KB) contains real `FIREBASE_SERVICE_ACCOUNT_KEY` (full Admin private key), `TWILIO_AUTH_TOKEN`, `SARVAM_AI_API_KEY`, `TAVILY_API_KEY`, `GEMINI_API_KEY`, `DEMO_PRINCIPAL_PASSWORD`.
- **Impact:** Anyone able to pull the builder image (registry access, `docker history`, layer export) recovers every secret. Next.js may also inline build-time env into the standalone bundle.
- **Fix:** Delete the `!.env.local` negation; never copy env files into any build stage; runtime secrets via Cloud Run env/Secret Manager. **Rotate all keys in `.env.local` now** if the image was ever pushed to a shared registry.

### C4. SSRF → GCP metadata / credential theft via image fetch
- **Where:** [image-utils.ts:14](../../src/ai/utils/image-utils.ts) — `fetch(imageUrl)` on a user-supplied URL, only a `data:` prefix check, no scheme/host/redirect/size guard. Reachable from [worksheet-wizard.ts:175](../../src/ai/flows/worksheet-wizard.ts) and [quiz-definitions.ts:110](../../src/ai/flows/quiz-definitions.ts) when `imageDataUri` is a URL.
- **Exploit:** `imageDataUri: "http://169.254.169.254/computeMetadata/v1/instance/service-accounts/default/token"` → server fetches GCP metadata, returns the token via the base64 payload path.
- **Fix:** Enforce `https:` + host allowlist (Firebase Storage / trusted CDNs only), block link-local/private ranges, disable redirects, cap body size.

### C5. Unauthenticated destructive DB endpoint
- **Where:** [migrate-ncert/route.ts:6](../../src/app/api/migrate-ncert/route.ts) — `GET` with zero auth/role check; any authenticated free user triggers a full `batch.set` reseed over `ncert_curriculum` (cost/DoS + clobbers curated curriculum).

### C6. School-wide data-export IDOR (latent mass-PII exfiltration)
- **Where:** [export/route.ts:40-49](../../src/app/api/export/route.ts) + [data-export-service.ts:471](../../src/lib/data-export-service.ts) — accepts `schoolExport:true` + `schoolNormalized` and creates a `school` export job with **no admin-of-that-school check**. - **Impact:** Any teacher can queue an export of every teacher's data for any school (processor not yet wired → latent, dangerous the moment it turns on).
- **Fix:** Verify caller is admin of `schoolNormalized`; implement real field stripping.

### C7. Two most expensive AI generators bypass all plan/quota limits
- **Where:** [lesson-plan/stream/route.ts:149](../../src/app/api/ai/lesson-plan/stream/route.ts) and [exam-paper/stream/route.ts:226](../../src/app/api/ai/exam-paper/stream/route.ts) export `POST = _handler` with **no `withPlanCheck`** (comment falsely claims otherwise). ### C8. Stored XSS on library download/print
- **Where:** [content-gallery.tsx:303-406](../../src/components/library/content-gallery.tsx) — `createDownloadableContent`/`formatQuiz`/`formatLessonPlan`/`formatRubric` interpolate stored/community-imported content into an HTML string with **no escaping**, then `window.open`/print ([:231](../../src/components/library/content-gallery.tsx)).
- **Exploit:** A saved or community-imported resource containing a `<script>` payload executes when the user downloads/prints it.

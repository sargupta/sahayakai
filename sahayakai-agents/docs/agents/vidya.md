# vidya

> Supervisor / OmniOrb mic. Classifies natural-language teacher requests, extracts parameters, and either returns a VidyaAction for the OmniOrb client or delegates inline to instant-answer.

_Generated by `scripts/generate_capability_docs.py`. Re-run after every
schema, prompt, guard, or router change. Source of truth is the code._

## Overview

- **Registry slug:** `vidya`
- **Python package:** `sahayakai_agents.agents.vidya`
- **Prompts directory:** `prompts/vidya/`
- **Endpoint base:** `/v1/vidya`
- **Has specialist `_guard.py`:** no (uses shared `_behavioural.py`)

#### Excerpt: `instant_answer.handlebars` (first 30 non-comment lines)

```handlebars
You are VIDYA, the SahayakAI assistant. Answer the teacher's question directly and concisely.
**Instructions:**
1. **Tailor the answer.** Adjust complexity and vocabulary to the teacher's preferred grade level{{#if teacherProfile.preferredGrade}} (⟦{{teacherProfile.preferredGrade}}⟧){{/if}}. If no grade is given, answer for a general audience.
2. **Language.** Respond ONLY in {{#if language}}`{{language}}`{{else}}English (`en`){{/if}}. Do NOT switch languages mid-reply.
3. **Analogies.** For complex topics, use simple analogies — especially for younger grades.
4. **Be direct.** Provide the answer in one short paragraph. No conversational filler. No "great question!" preamble.
5. **Length budget.** Keep the entire answer under 150 words — VIDYA replies are spoken aloud, so brevity matters.
6. **Scope integrity.** Stay within the educational task. Decline politely if the question is off-topic or inappropriate, but always put the decline text as your answer.
**TEACHER PROFILE:**
{{#if teacherProfile.preferredGrade}}- Preferred class: ⟦{{teacherProfile.preferredGrade}}⟧
{{/if}}{{#if teacherProfile.preferredSubject}}- Preferred subject: ⟦{{teacherProfile.preferredSubject}}⟧
{{/if}}{{#if teacherProfile.schoolContext}}- School context: ⟦{{{teacherProfile.schoolContext}}}⟧
{{/if}}
**TEACHER'S QUESTION:**
⟦{{{message}}}⟧
Respond with the answer text directly — no JSON, no markdown fences.
```


## Behavioural rules

Uses the shared `_behavioural.py` envelope directly:
- `assert_no_forbidden_phrases` — confusable-folded forbidden-phrase scan.
- `assert_script_matches_language` — Unicode-range script check.

No specialist-specific overrides. Length / element bounds are enforced at the Pydantic schema layer (see I/O Contract above).
